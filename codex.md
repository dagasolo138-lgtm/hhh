## 1. 项目定义

这是一个部署在 GitHub Pages 的纯前端、单标的中国黄金模拟市场。

唯一可交易资产：**人民币计价的“中国黄金现货指数基金”模拟份额**。它高拟合追踪境内黄金现货，交易方式类似场内基金或股票。

项目目标是构建可复盘的市场结构，不是随机涨跌演示器：

```text
国际黄金与汇率
→ 中国境内黄金价格与本地溢价
→ 基金理论净值
→ 交易者预期、订单与订单簿
→ 屏幕交易价格
→ 成交、情绪、风险和下一刻订单行为
```

当前数据全部由模拟世界生成。未来可以用真实公开数据校准初始价格、波动范围和参数，但模拟世界不能伪装成真实行情。

## 2. 不可破坏的世界规则

1. **理论净值与交易价格必须分离。**
   - 理论净值来自境内黄金现货、每份黄金含量、跟踪误差和费用拖累。
   - 二级市场交易价格来自订单簿撮合、流动性和溢价折价。

2. **中国黄金基础价格链必须完整。**

   ```text
   国际现货黄金（美元/盎司）
   × 美元兑人民币
   ÷ 31.1034768
   = 人民币换算金价（人民币/克）

   人民币换算金价 ×（1 + 国内黄金溢价）
   = 国内黄金现货价格

   国内黄金现货价格 × 每份黄金含量
   ×（1 + 跟踪误差）×（1 - 管理费拖累）
   = 基金理论净值
   ```

3. **技术指标不能直接改价格。**
   均线、动量、波动带和成交量只能影响趋势资金等交易者的订单；价格必须通过订单簿和成交形成。

4. **本刻结果只能在下一刻反馈。**
   收益、成交量、订单失衡、回撤和波动率必须在下一刻才影响情绪、风险预算和后续订单。禁止同一刻自我循环。

5. **同一种子必须可复现。**
   相同初始状态、相同随机种子和相同调用顺序，应生成同一条市场世界线。

6. **现金和份额必须守恒。**
   自动交易者、做市商和玩家之间的现金与基金份额必须有明确去向。

7. **玩家不能直接看见隐藏状态。**
   乐观度、注意力、分歧度、脆弱度、风险偏好和信息不对称属于后台状态；页面只展示公开数据和推断摘要。

8. **智能分析只能读取。**
   智能分析不能写入价格、随机种子、订单簿、玩家账户或存档。

9. **重要变化必须有日志。**
   市场推进、公开事件、玩家成交和状态摘要必须可回放。

10. **静态网页没有服务器数据库。**
    同浏览器本地存档可恢复；跨设备、清空数据或无痕模式不保证保留。重要世界线必须导出。

## 5. 关键状态结构

```text
状态 = {
  版本: 2,
  资产,
  系统,
  变量,
  派生,
  交易者,
  订单簿,
  事件,
  结构,
  历史
}
```

### 5.1 `资产`

- 名称：`中国黄金现货指数基金`
- 简称：`中国黄金`
- 计价货币：人民币
- 最小交易单位：100 份
- 价格最小变动：0.001
- 每份黄金含量：当前默认 0.01 克/份

真实标的校准前，必须先确认产品机制、每份含金量和交易单位。

### 5.2 `系统`

- `随机种子`：决定世界线可复现性。
- `演算序号`：每推进一刻加一。
- `虚拟时间`：模拟时钟。
- `交易日序号`：低频事件和季节性基准。
- `盘中阶段`：集合竞价、上午交易、午间休市、下午交易、收盘后、休市。
- `国际市场阶段`：亚洲、伦敦、纽约、重叠、国际休市。
- `是否可交易`：玩家订单先检查该字段。
- `守恒基线`：自动交易者与玩家合计现金、份额的初始总额。

### 5.3 `变量`

所有新变量必须先登记在 `变量字典.js`。主要分组：

- 全球黄金：国际现货黄金、实际利率冲击、美元强弱、通胀预期、地缘风险、全球资金流、条件方差。
- 汇率与中国映射：美元兑人民币、人民币换算金价、国内黄金溢价、国内黄金现货、期现基差。
- 基金：每份黄金含量、理论净值、交易价格、溢价折价、跟踪误差、申赎摩擦、套利压力。
- 情绪与风险：乐观度、注意力、分歧度、脆弱度、风险偏好、信息不对称、情绪不确定度。
- 微观结构：买卖深度、价差、撤单率、补单率、主动买卖、订单失衡、价格冲击系数、实现波动率。
- 玩家账户：玩家现金、持仓、成本、市值、浮动盈亏、总资产。

### 5.4 `派生`

常见派生字段：

- `本地流动性状态`
- `开盘前流动性`
- `国内黄金溢价变化`
- `双向主动成交量`
- `订单失衡反复程度`
- `市场状态摘要`

派生值不应被当作世界根变量。

### 5.5 `交易者`

| 标识 | 作用 |
|---|---|
| `价值套利` | 根据交易价与理论净值偏离形成均值回归订单。 |
| `趋势资金` | 根据技术指标、风险偏好与波动率形成趋势订单。 |
| `散户群体` | 根据乐观度、注意力、叙事与回撤形成净需求。 |
| `风险控制资金` | 根据回撤、脆弱度和波动率减仓。 |
| `信息优势资金` | 根据带噪声的提前信号下单。 |
| `做市商` | 提供双边盘口，承接成交并积累库存。 |

### 5.6 `订单簿`

```text
订单簿 = {
  买方: [{ 档位, 价格, 数量 }, ...10档],
  卖方: [{ 档位, 价格, 数量 }, ...10档],
  中间价,
  上次成交价,
  版本
}
```

自动交易者和玩家市价单都可以吃单。限价委托和挂单队列尚未实现。

### 5.7 `历史`

- `价格序列`：价格、成交量、成交额、理论净值、溢价折价。
- `日志`：市场推进、公开事件、玩家成交。
- `事件`：公开事件历史。
- `状态快照`：定期摘要，供回放和智能分析使用。

## 6. 一刻市场如何推进

执行入口：`市场/时序演算器.js` 的 `推进时序中国黄金市场`。

时序规则：先读取上一刻观测，运行本刻市场，再把本刻结果写入下一刻观测。

当前演算顺序：

1. 推进虚拟时间，判定中国交易阶段与国际市场阶段。
2. 更新事件系统：新事件、衰减、公开延迟、泄露强度、叙事和热度。
3. 更新全球黄金与汇率。
4. 更新中国本地黄金、理论净值、期现基差和套利压力。
5. 更新隐藏心理状态。
6. 形成交易者目标仓位与订单。
7. 构建十档订单簿，计算价差、深度、撤单率、补单率和冲击系数。
8. 撮合自动交易者订单，形成主动买卖量、订单失衡、成交量、成交额和做市库存。
9. 更新交易价格、收益、波动率、回撤、溢价折价和技术指标。
10. 更新成本分布、日志、快照和跨期反馈。
11. 写入下一刻观测，并执行一致性检查。

概念上可称“八阶段”；代码为了可维护性拆成更多函数，不能改变时序。

## 7. 同刻因果与跨期反馈

### 同刻因果

```text
时间制度
→ 全球环境
→ 国内映射
→ 事件与心理
→ 交易者订单
→ 订单簿撮合
→ 市场统计
→ 存档与反馈
```

### 跨期反馈

以下关系必须读取上一刻观测：

- 价格收益 → 乐观度。
- 成交量、异常成交量 → 注意力。
- 双向主动成交、订单失衡反复、波动率 → 分歧度。
- 订单簿脆弱度、回撤 → 脆弱度。
- 持仓成本分布 → 最近盈利体验 → 风险偏好。
- 回撤、波动率 → 风险控制减仓压力。
- 技术指标集合 → 下一刻趋势资金目标仓位。
- 成交结果、订单失衡 → 下一刻做市库存与报价意愿。
- 中国休市期间的国际黄金、汇率、本地溢价变化 → 隔夜信息积累 → 下一开盘跳空压力。

时序权威文件：`市场/时序因果图.js`。

`因果图.js` 是较早的全量概念图和查询工具，不应作为当前运行时序的唯一依据。新增时序规则时，优先更新 `市场/时序因果图.js` 和 `市场/时序校验.js`。

## 10. 存档与兼容性

### 存档现状

- 主存储：浏览器 IndexedDB。
- 备用：浏览器 localStorage。
- 同设备、同浏览器正常刷新或重开：通常可恢复。
- 跨设备、清除站点数据、无痕模式：可能丢失。
- 导出 JSON：重要世界线的可靠迁移方式。

### 版本规则

- 当前中国黄金市场状态版本：`2`。
- 旧版华辰工业状态不能直接导入版本 2 市场。
- 页面遇到旧存档时会新建中国黄金世界线，并提示用户保留旧导出文件。

### 已知命名债务

`存档.js` 内的 IndexedDB 名称和备用键仍含旧“华辰工业”名称：

```text
华辰工业市场存档
华辰工业市场存档备份
```

功能当前可用，但名称与项目不一致。迁移时必须同时处理旧键读取、版本 2 数据迁移和用户世界线保护。

## 11. 验证与测试

### 本地自检

执行：

```text
npm run 自检
```

自检至少应验证：

- 同种子可复现。
- 不同种子不会完全相同。
- 连续推进后交易价格、理论净值、盘口均有效。
- 买一价低于卖一价。
- 玩家可以买入再卖出。
- 玩家成交后现金和份额仍守恒。
- 一致性检查通过。

### GitHub Actions

工作流文件：`.github/workflows/市场自检.yml`。

当前代码已配置工作流，但本对话尚未拿到可靠云端执行结果。不得把“已配置”写成“已通过”；必须实际查看工作流结果后再记录。

### 改市场逻辑后的最低检查清单

1. 新变量是否先登记到 `变量字典.js`。
2. 新时序关系是否先登记到 `市场/时序因果图.js`。
3. 是否出现同一刻自我循环。
4. 是否破坏理论净值公式。
5. 是否破坏现金或份额守恒。
6. 是否新增无法解释的直接价格修改。
7. 是否能写入完整日志。
8. 是否在本文件末尾追加版本记录。

## 12. 智能分析与安全边界

当前 `智能分析.js` 只构建资料和请求边界，不直接连接外部服务。

未来接入 DeepSeek 或其他模型时：

- 页面端不得硬编码密钥。
- 需要安全服务端或受控代理。
- 模型只能收到公开资料、允许的推断摘要和波动日志。
- 模型输出必须标明事实、推断和无法判断部分。
- 模型不得修改市场状态、随机种子、订单簿、玩家账户或存档。

## 13. 推荐开发顺序

1. 限价单、撤单、订单队列与时间优先撮合。
