> 本文件是项目长期交接中枢。开发时按需要查阅；**每次开发完成后，必须在本文末尾追加一条版本更新概述。**

---

## 1. 项目定义

这是一个部署在 GitHub Pages 的纯前端、单标的中国黄金模拟市场。

唯一资产：**人民币计价的中国黄金现货ETF模拟份额**。制度框架锁定为**上交所黄金ETF通用制度**；项目不绑定某一只现实基金代码。

```text
国际现货黄金（美元/盎司）
× 美元兑人民币
÷ 31.1034768
→ 人民币换算金价（人民币/克）
→ 国内黄金溢价
→ 国内黄金现货价格
→ 基金理论净值
→ 自动交易者目标仓位
→ 中心限价订单簿逐笔成交
→ 二级市场交易价格
```

项目用于市场机制模拟与教育，不连接真实交易，不构成投资建议。

---

## 2. 不可破坏的规则

1. 理论净值与二级市场交易价格必须分离。
2. 技术指标只能影响交易者订单，不能直接改价格。
3. 本刻结果只能在下一刻反馈，禁止同刻自循环。
4. 同初始状态、同随机种子、同调用顺序必须可复现。
5. 自动交易者、做市商与玩家之间的现金和份额必须守恒。
6. 隐藏情绪变量不能直接暴露给玩家。
7. 智能分析只能读取，不能改市场状态、订单簿、账户或存档。
8. 市场推进、成交、委托、撤单、过期、集合竞价开盘和自动订单流都必须写入可追溯记录。
9. 二级市场交易、现金申赎和黄金现货合约申赎必须分开实现。
10. 静态网页只有浏览器本地存档；重要世界线必须导出 JSON。
11. 中心订单簿中的订单必须满足价格优先、同价时间优先和自成交保护。

## 3. 交易制度与当前实现

权威文件：

- `交易制度参数表.md`：面向人阅读的制度说明。
- `市场/交易制度参数.js`：机器可读取的制度参数唯一来源。
- 最新交易所规则、基金合同、招募说明书、申赎清单和当日公告优先于项目内文档。

### 已执行

| 规则 | 当前实现 |
|---|---|
| 最小报价单位 | 0.001 元；委托、盘口、交易价和开盘价均对齐。 |
| 买入数量 | 100 份起，必须为 100 份整数倍。 |
| 零股卖出 | 支持。 |
| 当日回转 | 当日买入可当日卖出。 |
| 涨跌幅 | 前收盘价 ±10%，约束市价单、限价单、盘口和开盘价。 |
| 连续竞价 | 09:30–11:30、13:00–15:00；自动中心撮合、玩家市价单、玩家限价单和撤单。 |
| 开盘集合竞价 | 09:15–09:25；独立订单池、09:20 后撤单锁定、09:30 单一开盘价。 |
| 自动交易者 | 五类自动资金与做市商进入中心限价订单簿真实清算。 |
| 限价有效期 | 玩家限价单当日有效；跨日自动过期并释放冻结资产。 |
| 可见十档 | 中心做市深度、自动订单残余、玩家挂单叠加展示并标明来源。 |

### 集合竞价生命周期

```text
09:15–09:20
  玩家限价单可提交、可撤销

09:20–09:25
  玩家限价单可提交，撤单锁定

09:30
  最大可成交量
  → 最小未成交量
  → 最接近前收盘价
  → 单一开盘价
  → 已成交部分按开盘价成交
  → 未成交部分转入连续竞价队列
```

自动中心撮合不在集合竞价期间成交；09:30 先完成集合竞价，自动订单从后续连续竞价刻开始进入中心簿。

### 当前双路径边界

- **自动交易者和做市商**：已经走中心限价订单簿，真实逐笔清算、真实改变交易价格。
- **玩家市价单和玩家连续竞价限价单**：仍走既有基础做市盘口路径，保持现有冻结资产、集合竞价和撤单机制。
- **可见十档**：读取中心盘口快照，再叠加玩家挂单；玩家挂单当前只展示，不作为同一玩家自身可成交对手方。

这套双路径是过渡架构。下一阶段必须让玩家订单也进入中心订单簿，删除基础盘口直接成交路径。

### 尚未实现

- 玩家市价单、玩家连续竞价限价单迁入中心订单簿。
- 多玩家账户、玩家之间订单交叉、同价比例分配。
- 收盘集合竞价。
- 真实中国交易日历、停复牌、异常状态和特殊首日规则。
- 现金申赎、黄金现货合约申赎、清单、额度、清算和交收。
- 手续费、税费、融资、保证金和真实公开数据校准。

## 7. 自动中心撮合规则

1. 旧引擎先更新宏观、国内金价、基金净值、隐藏心理、自动交易者目标仓位和基础做市报价。
2. 接管器将自动交易者的旧聚合成交上限暂时设为零，防止旧内核重复成交。
3. 中心订单簿根据目标仓位差额、原单刻上限、现金和持仓约束生成自动订单。
4. 每个自动资金订单拆为主动限价部分和被动限价部分；做市商则提交十档双边报价。
5. 中心订单簿按买方价格从高到低、卖方价格从低到高、同价时间先后撮合。
6. 成交价采用较早进入订单簿的一方价格。
7. 成交更新买卖双方现金、持仓、平均成本和已实现盈亏。
8. 基金交易价格取最后一笔中心成交价，并更新收益、成交量、成交额、订单失衡、波动和中心深度。
9. 未成交自动订单本刻可见；下一刻刷新时按“本刻有效”过期并进入订单历史。
10. 同一参与者买卖订单相遇时触发自成交保护，较晚订单被拒绝。

## 8. 一刻市场推进顺序

1. 初始化制度状态、守恒基线和上一刻观测。
2. 记录推进前中国交易日。
3. 旧引擎推进：时间、事件、全球黄金、汇率、国内金价、净值、心理、目标仓位、基础做市报价。
4. 非集合竞价、非09:30开盘切换时，中心订单簿撮合自动订单与做市报价。
5. 使跨交易日玩家委托失效。
6. 制度收尾：刷新日线边界、限制交易价、规范基础做市盘口。
7. 若从集合竞价切换至上午连续竞价，执行 09:30 开盘集合竞价。
8. 在基础做市盘口中撮合玩家连续竞价限价队列。
9. 同步统一委托流，叠加中心盘口与玩家挂单。
10. 写入上一刻观测，执行市场一致性与制度一致性检查。

## 11. 自检

命令：

```text
npm run 自检
```

当前命令依次运行：

```text
市场/自检.js
市场/统一委托流自检.js
市场/中心订单簿自检.js
```

当前自检目标：

- 同种子复现；不同种子不同。
- 集合竞价拒绝市价单。
- 集合竞价可提交、09:20 后锁定撤单、09:30 形成开盘价。
- 集合竞价订单不丢失，成交或转连续竞价。
- 连续竞价同价订单先到先成交。
- 部分成交后冻结资产正确。
- 撤单和跨日过期后冻结资产释放。
- 交易价与盘口不突破涨跌停边界。
- 现金、份额、订单数量与冻结资产保持一致。
- 自动订单统一格式、可见十档来源叠加、玩家挂单不可自成交、可见深度不改写基础做市盘口。
- 自动中心订单簿逐笔成交、成交价格形成、自动参与者现金份额守恒、自成交保护和中心盘口排序。

GitHub Actions 工作流已配置，但当前对话尚未取得可靠云端执行结果。不得写成“云端已通过”。
