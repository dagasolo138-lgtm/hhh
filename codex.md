## 1. 项目定义

这是一个部署在 GitHub Pages 的纯前端、单标的中国黄金模拟市场。

唯一可交易资产：**人民币计价的中国黄金现货ETF模拟份额**。

当前锁定的制度框架：**上海证券交易所黄金ETF通用制度**。项目暂不绑定某一只具体基金代码；未来选择真实参考产品后，再单独校准每份含金量、管理费、申赎单位、基金规模与初始数据。

市场主链：

```text
国际现货黄金（美元/盎司）
× 美元兑人民币
÷ 31.1034768
→ 人民币换算金价（人民币/克）
→ 国内黄金溢价
→ 国内黄金现货价格
→ 基金理论净值
→ 交易者订单与订单簿撮合
→ 二级市场交易价格
→ 成交、情绪、风险和下一刻订单行为
```

目标是构建可复盘的市场机制，不是随机涨跌演示器，也不连接真实交易。

## 2. 不可破坏的世界规则

1. **理论净值与交易价格必须分离。**
   - 理论净值来自境内黄金现货、每份黄金含量、跟踪误差与费用拖累。
   - 二级市场交易价格来自订单簿、成交、流动性和溢价折价。

2. **技术指标不能直接改价格。**
   均线、动量、成交量和波动带只能影响交易者的订单与目标仓位。

3. **本刻结果只能在下一刻反馈。**
   收益、成交量、订单失衡、回撤和波动率不能回头影响本刻已经形成的订单。

4. **同一种子必须可复现。**
   相同初始状态、随机种子和调用顺序应得到同一条市场世界线。

5. **现金与基金份额必须守恒。**
   自动交易者、做市商和玩家之间每一笔资金与份额转移必须有对手方。

6. **玩家不能直接看到隐藏状态。**
   乐观度、注意力、分歧度、脆弱度、风险偏好和信息不对称属于后台状态；页面只展示公开数据和推断摘要。

7. **智能分析只能读取。**
   智能分析不能改价格、随机种子、订单簿、账户、存档或交易制度。

8. **重要变化必须有日志。**
   市场推进、公开事件、玩家成交、订单状态和制度拒绝原因都应可回放。

9. **二级市场交易、现金申赎、黄金现货合约申赎必须分开。**
   它们的资格、时段、交收和可用份额规则不同，禁止用同一个“买卖”流程混合实现。

10. **静态网页没有服务器数据库。**
    本地存档只能在同浏览器环境中通常恢复；重要世界线必须导出。

## 3. 交易制度基准

权威文件：

- `交易制度参数表.md`：面向人阅读的制度表、已实现状态与待实现缺口。
- `市场/交易制度参数.js`：机器可读取的制度参数唯一来源。
- `市场/制度执行器.js`：制度参数进入市场状态、订单验证、价格边界、冻结资产的执行层。
- `市场/制度收尾.js`：市场推进后刷新交易日边界、限制价格、规范盘口、累计当日统计。
- `市场/制度一致性检查.js`：检查涨跌停、报价单位、盘口边界、冻结资产和订单容器。

当前锁定的上交所黄金ETF二级市场规则：

| 规则 | 参数 | 当前实现状态 |
|---|---:|---|
| 当日回转 | 当日买入可当日卖出 | 已实现市价买卖 |
| 最小报价单位 | 0.001元 | 已由制度执行层统一读取 |
| 买入最小数量 | 100份 | 已执行 |
| 买入数量 | 100份整数倍 | 已执行 |
| 零股卖出 | 允许 | 市价卖出已允许；限价卖出待实现 |
| 涨跌幅 | 相对前收盘价 ±10% | 已接入价格、盘口和玩家市价成交 |
| 集合竞价 | 09:15–09:25 | 已禁止即时市价成交；独立订单池待实现 |
| 连续竞价 | 09:30–11:30、13:00–15:00 | 已作为即时市价成交的唯一时段 |
| 大宗交易 | 15:00–15:30 | 当前不模拟 |

黄金ETF申赎规则：

- 现金申赎：09:30–11:30、13:00–15:00；属于独立业务，不作为普通玩家交易按钮。
- 黄金现货合约申赎：09:30–11:30、13:00–15:00；需要证券账户、黄金账户、备案和相应会员通道。
- 普通模拟玩家暂不开放两类申赎；未来只作为授权参与者或套利资金的后台机制。

### 当前制度缺口

当前页面仍不能宣称“完全复刻真实上交所黄金ETF”，因为仍缺：

- 独立集合竞价订单池和开盘单一价格算法。
- 限价单、撤单、价格优先、时间优先队列。
- 真实交易日历、停复牌、异常状态、申赎清单、额度和结算。
- 具体基金合同参数和真实公开数据校准。
- 自动交易者的订单在生成阶段直接进入撮合，尚未完全复用未来的统一限价委托队列。

## 8. 一刻市场如何推进

执行入口：`市场/时序演算器.js` 的 `推进时序中国黄金市场`。

规则：先读取上一刻观测，运行本刻市场，再写入下一刻观测。

当前顺序：

1. 初始化或修复制度状态、守恒基线和上一刻观测。
2. 记录当前中国交易日，进入市场主引擎。
3. 推进虚拟时间，判定中国交易阶段和国际市场阶段。
4. 更新事件系统、全球黄金与汇率。
5. 更新中国本地黄金、理论净值、期现基差与套利压力。
6. 更新隐藏心理状态，形成交易者订单。
7. 构建十档订单簿，撮合自动交易者订单。
8. 更新交易价格、收益、波动率、回撤、溢价折价与技术指标。
9. 制度收尾：刷新交易日、限制交易价格、规范盘口、累计当日成交。
10. 写入上一刻观测，并执行市场一致性与制度一致性检查。

概念上可称八阶段；代码为了可维护性拆成更多函数，不能改变时序。

## 9. 同刻因果与跨期反馈

同刻因果：

```text
时间制度
→ 全球环境
→ 国内映射
→ 事件与心理
→ 交易者订单
→ 订单簿撮合
→ 市场统计
→ 存档与反馈
```

必须读取上一刻观测的反馈：

- 价格收益 → 乐观度。
- 成交量、异常成交量 → 注意力。
- 双向主动成交、订单失衡反复、波动率 → 分歧度。
- 订单簿脆弱度、回撤 → 脆弱度。
- 持仓成本分布 → 最近盈利体验 → 风险偏好。
- 回撤、波动率 → 风险控制减仓压力。
- 技术指标集合 → 下一刻趋势资金目标仓位。
- 成交结果、订单失衡 → 下一刻做市库存与报价意愿。
- 中国休市期间的国际黄金、汇率、本地溢价变化 → 隔夜信息积累 → 下一开盘跳空压力。

运行时序权威文件是 `市场/时序因果图.js`。

## 10. 存档与兼容性

- 主存储：浏览器 IndexedDB。
- 备用：浏览器 localStorage。
- 同设备、同浏览器正常刷新或重开：通常可恢复。
- 跨设备、清除站点数据、无痕模式：可能丢失。
- 导出JSON：重要世界线的可靠迁移方式。

当前中国黄金市场状态版本：`2`。

此次改动只新增可选结构字段。旧版版本2本地存档在加载后会由制度执行器补齐制度状态、冻结资产和订单容器；不需要升版。

旧版华辰工业状态不能直接导入版本2市场。

已知命名债务：`存档.js` 内的数据库名称和备用键仍含“华辰工业”。功能可用，但迁移时必须保护旧世界线并处理旧键读取。

## 11. 验证与测试

本地自检：

```text
npm run 自检
```

自检当前应验证：

- 同种子可复现。
- 不同种子不会完全相同。
- 连续推进后交易价格、理论净值和盘口有效。
- 买一价低于卖一价。
- 集合竞价阶段不得发生即时市价成交。
- 非100份整数倍买入必须拒绝。
- 玩家可以买入再卖出。
- 玩家成交后现金与份额守恒。
- 交易价和盘口不越过当日涨跌停边界。
- 冻结现金和冻结份额在未实现限价单前保持零。
- 市场一致性与制度一致性检查通过。

GitHub Actions 工作流：`.github/workflows/市场自检.yml`。

当前已配置工作流，但本对话尚未拿到可靠云端执行结果。不得把“已配置”写成“已通过”。

## 12. 智能分析与安全边界

当前 `智能分析.js` 只构建资料和请求边界，不直接连接外部服务。

未来接入 DeepSeek 或其他模型时：

- 页面端不得硬编码密钥。
- 需要安全服务端或受控代理。
- 模型只能读取公开资料、允许的推断摘要和波动日志。
- 模型输出必须区分事实、推断与无法判断。
- 模型不得修改市场状态、随机种子、订单簿、玩家账户或存档。
- 模型不得输出针对现实资产的个性化买卖指令。
