# 中国黄金模拟市场 · Codex

> 本文件是项目长期交接中枢。开发时按需要查阅；**每次开发完成后，必须在本文末尾追加一条版本更新概述。**

---

## 1. 项目定义

这是一个部署在 GitHub Pages 的纯前端、单标的中国黄金模拟市场。

唯一资产：**人民币计价的中国黄金现货ETF模拟份额**。制度框架锁定为**上交所黄金ETF通用制度**；项目不绑定某一只现实基金代码。

```text
国际现货黄金（美元/盎司）
× 美元兑人民币
÷ 31.1034768
→ 人民币换算金价（人民币/克）
→ 国内黄金溢价
→ 国内黄金现货价格
→ 基金理论净值
→ 自动交易者目标仓位 + 玩家订单
→ 开盘集合竞价 / 连续竞价中心订单簿
→ 二级市场交易价格
```

项目用于市场机制模拟与教育，不连接真实交易，不构成投资建议。

---

## 2. 不可破坏的规则

1. 理论净值与二级市场交易价格必须分离。
2. 技术指标只能影响交易者订单，不能直接改价格。
3. 本刻结果只能在下一刻反馈，禁止同刻自循环。
4. 同初始状态、同随机种子、同调用顺序必须可复现。
5. 自动交易者、做市商与玩家之间的现金和份额必须守恒。
6. 隐藏情绪变量不能直接暴露给玩家。
7. 智能分析只能读取，不能改市场状态、订单簿、账户或存档。
8. 市场推进、成交、委托、撤单、过期、集合竞价开盘与中心订单流必须写入可追溯记录。
9. 二级市场交易、现金申赎和黄金现货合约申赎必须分开实现。
10. 静态网页只有浏览器本地存档；重要世界线必须导出 JSON。
11. 中心订单簿中的订单必须满足价格优先、同价时间优先和自成交保护。
12. 玩家限价单的冻结现金或份额必须与待处理订单的剩余数量精确一致。

## 3. 当前交易架构

### 3.1 开盘集合竞价

```text
09:15–09:20  玩家限价单可提交、可撤销
09:20–09:25  玩家限价单可提交，撤单锁定
09:30          单一开盘价
               已成交部分按开盘价成交
               未成交部分转入连续竞价中心订单簿
```

- 集合竞价只接受限价单。
- 开盘价选择顺序：最大可成交量 → 最小未成交量 → 最接近前收盘价 → 最小报价单位。
- 当前集合竞价由玩家订单与自动做市背景深度形成。
- 自动交易者尚未进入开盘集合竞价订单池。

### 3.2 连续竞价统一中心订单簿

连续竞价中的以下参与者共用 `市场/中心订单簿.js`：

```text
做市商双边报价
+ 价值套利
+ 趋势资金
+ 散户群体
+ 风险控制资金
+ 先行信号资金
+ 玩家市价单
+ 玩家限价单
```

规则：

```text
买单：价格高者优先；同价中心委托序号早者优先。
卖单：价格低者优先；同价中心委托序号早者优先。
```

- 玩家市价单以涨跌停边界形成即时有效中心订单；未成交部分当刻失效。
- 玩家限价单冻结资金或份额后进入中心订单簿；剩余部分当日有效，跨日失效。
- 玩家限价单跨刻不会被自动过期；每一刻会从玩家待处理容器重新注入中心簿。
- 同一玩家账户的交叉订单不会成交，后提交订单会被拒绝。
- 中心盘口快照已包含做市、自动和玩家挂单；可见十档不能二次叠加玩家数量。

## 4. 状态结构与关键容器

```text
状态 = {
  版本: 2,
  资产,
  系统,
  变量,
  派生,
  交易者,
  订单簿,
  事件,
  结构,
  历史
}
```

### `结构.交易制度`

```text
{
  当前交易日,
  上一交易日收盘价,
  当日涨停价,
  当日跌停价,
  当日开盘价,
  当日最高价,
  当日最低价,
  当日成交量,
  当日成交额,
  当前时分,
  集合竞价中,
  集合竞价可撤单,
  连续竞价中
}
```

### `结构.玩家冻结资产`

```text
{ 现金, 份额 }
```

- 买单冻结：`限价 × 剩余数量`。
- 卖单冻结：`剩余数量`。
- 成交、撤单、过期和拒绝都必须释放对应冻结资产。

### `结构.玩家委托`

```text
{ 下次编号, 待处理, 历史, 状态统计 }
```

连续竞价玩家限价单在此处保存为长期订单本体；中心订单簿持有同一订单对象的当刻映射。

### `结构.集合竞价`

```text
{
  当前交易日,
  待处理,
  历史,
  最近结果,
  最近开盘价,
  最近成交量,
  最近成交额
}
```

### `结构.中心订单簿`

```text
{
  下次委托序号,
  本刻订单,
  本刻成交,
  订单历史,
  成交历史,
  盘口快照,
  本刻自动意图,
  版本,
  最近更新
}
```

中心订单关键字段：

```text
编号、中心委托序号、来源、参与者标识、订单类别、方向、价格、数量、
已成交数量、剩余数量、状态、有效期、做市报价、自动生成、玩家订单、市价单
```

订单状态：`待撮合`、`部分成交`、`全部成交`、`已撤销`、`已拒绝`、`已过期`。

## 5. 运行时序

```text
1. 初始化制度状态、守恒基线和上一刻观测。
2. 旧市场引擎推进宏观、事件、黄金、汇率、净值、心理、目标仓位和基础做市报价。
3. 暂停旧引擎的自动聚合成交。
4. 结束上一刻自动/做市本刻订单；玩家当日有效订单保留在玩家容器。
5. 连续竞价阶段：自动订单、做市报价和玩家待处理限价单进入中心订单簿并逐笔撮合。
6. 跨日玩家订单失效。
7. 制度收尾：刷新时间、涨跌停、基础报价、日内统计和可观测深度。
8. 若 09:30 从集合竞价切换到上午交易，执行开盘集合竞价。
9. 同步玩家限价订单的中心成交结果、冻结资产、历史和日志。
10. 同步自动委托流与可见十档，写入上一刻观测并做一致性检查。
```

### 关键边界

- 自动中心撮合只在连续竞价执行。
- 09:30 优先完成开盘集合竞价；自动订单从后续连续竞价刻开始进入中心簿。
- 非连续竞价不保留陈旧的自动/做市中心挂单快照。
- 日内成交累计由中心成交或制度收尾择一登记，避免重复累计。

## 7. 对外接口

页面通过 `核心.js` 调用：

```text
创建新市场(随机种子)
推进一步(状态)
玩家市价交易(状态, 方向, 数量)
玩家限价交易(状态, 方向, 数量, 价格)
撤销限价委托(状态, 编号)
读取界面数据(状态)
创建分析资料(状态, 最近条数)
格式化市场时间(时间)
```

`读取界面数据` 包含：

```text
市场：前收盘、涨跌停、竞价状态、当前时分
账户：可用与冻结现金、可用与冻结持仓
委托：集合与连续委托、状态统计、最近集合竞价结果
订单簿：做市、自动、玩家三类来源的可见十档
自动委托：本刻中心自动订单
```

## 8. 存档与兼容性

- 市场状态版本仍为 `2`。
- 中心订单簿、统一委托流与玩家中心字段采用懒初始化；旧中国黄金版本 2 存档可加载后补齐新结构。
- 旧华辰工业存档不能直接进入中国黄金引擎。
- `存档.js` 的 IndexedDB 与备用键仍含旧“华辰工业”名称；迁移时必须保护旧世界线。

## 9. 自检

命令：

```text
npm run 自检
```

当前命令依次运行：

```text
市场/自检.js
市场/统一委托流自检.js
市场/中心订单簿自检.js
市场/玩家中心订单自检.js
```

应覆盖：

- 同种子复现；不同种子不同。
- 集合竞价撤单窗口、锁单、开盘价和转连续竞价。
- 自动和做市订单价格时间优先、守恒和自成交保护。
- 玩家市价单通过中心订单簿成交。
- 玩家限价单冻结、中心映射、跨刻保留、撤单与冻结释放。
- 涨跌停、报价单位、十档来源拆分、现金份额守恒。

GitHub Actions 工作流已配置，但当前对话未取得可靠的直推运行结果。不得写成“云端已通过”。
