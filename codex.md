> 本文件是项目长期交接中枢。开发时按需要查阅；**每次开发完成后，必须在本文末尾追加版本更新概述。**

---

## 1. 项目定义

这是一个部署在 GitHub Pages 的纯前端、单标的中国黄金模拟市场。

唯一资产：**人民币计价的中国黄金现货ETF模拟份额**。制度框架锁定为**上交所黄金ETF通用制度**；项目暂不绑定某一只现实基金代码。

```text
国际现货黄金（美元/盎司）
× 美元兑人民币
÷ 31.1034768
→ 人民币换算金价（人民币/克）
→ 国内黄金溢价
→ 国内黄金现货价格
→ 基金理论净值
→ 自动交易者与玩家订单
→ 订单簿、集合竞价与连续竞价
→ 二级市场交易价格
```

项目用于市场机制模拟与教育，不连接真实交易，不构成投资建议。

---

## 2. 不可破坏的规则

1. 理论净值与二级市场交易价格必须分离。
2. 技术指标只能影响交易者订单，不能直接改价格。
3. 本刻结果只能在下一刻反馈，禁止同刻自循环。
4. 同初始状态、同随机种子、同调用顺序必须可复现。
5. 自动交易者、做市商与玩家之间的现金和份额必须守恒。
6. 隐藏情绪变量不能直接暴露给玩家。
7. 智能分析只能读取，不能改市场状态、订单簿、账户或存档。
8. 市场推进、成交、委托、撤单、过期和集合竞价开盘都必须写日志。
9. 二级市场交易、现金申赎和黄金现货合约申赎必须分开实现。
10. 静态网页只有浏览器本地存档；重要世界线必须导出 JSON。

## 3. 交易制度与当前实现

权威文件：

- `交易制度参数表.md`：面向人阅读的制度说明。
- `市场/交易制度参数.js`：机器可读取的制度参数唯一来源。
- 最新交易所规则、基金合同、招募说明书、申赎清单和当日公告优先于项目内文档。

### 已执行

| 规则 | 当前实现 |
|---|---|
| 最小报价单位 | 0.001 元；委托、盘口、交易价和开盘价均对齐。 |
| 买入数量 | 100 份起，必须为 100 份整数倍。 |
| 零股卖出 | 支持。 |
| 当日回转 | 当日买入可当日卖出。 |
| 涨跌幅 | 前收盘价 ±10%，约束市价单、限价单、盘口和开盘价。 |
| 连续竞价 | 09:30–11:30、13:00–15:00；市价单、限价单、撤单、价格优先、同价时间优先。 |
| 开盘集合竞价 | 09:15–09:25；独立订单池、09:20 后撤单锁定、09:30 单一开盘价。 |
| 限价单有效期 | 当日有效；跨日自动过期并释放冻结资产。 |

### 集合竞价生命周期

```text
09:15–09:20
  限价单可提交、可撤销

09:20–09:25
  限价单可提交，撤单锁定

09:30
  最大可成交量
  → 最小未成交量
  → 最接近前收盘价
  → 单一开盘价
  → 已成交部分按开盘价成交
  → 未成交部分转入连续竞价队列
```

### 尚未实现

- 真实中国交易日历、停复牌、异常状态和特殊首日规则。
- 收盘集合竞价。
- 自动交易者进入统一限价委托队列。
- 多玩家账户、玩家之间订单交叉与同价比例分配。
- 现金申赎、黄金现货合约申赎、清单、额度、清算和交收。
- 手续费、税费、融资、保证金和真实公开数据校准。

## 7. 委托规则

### 连续竞价

```text
买单：价格高者优先；同价先提交者优先。
卖单：价格低者优先；同价先提交者优先。
```

限价买单只吃卖方价格不高于限价的深度；限价卖单只吃买方价格不低于限价的深度。

### 集合竞价

- 只接受限价单，不接受市价单。
- 玩家订单与自动做市背景深度共同形成候选开盘价。
- 所有集合竞价成交统一使用一个开盘价。
- 未成交部分保留原委托序号、剩余数量和冻结资产，转入连续竞价。
- 当前只有一个玩家账户；玩家集合竞价单主要与自动做市背景深度成交。

## 8. 一刻市场推进顺序

1. 初始化制度状态、守恒基线和上一刻观测。
2. 自动市场推进：时间、事件、全球黄金、汇率、国内金价、净值、心理、自动订单、自动盘口、自动成交和统计。
3. 使跨交易日玩家委托失效。
4. 制度收尾：刷新日线边界、限制交易价、规范盘口。
5. 若从集合竞价切换至上午连续竞价，执行 09:30 开盘集合竞价。
6. 在连续竞价盘口中撮合玩家限价队列。
7. 写入上一刻观测，执行市场一致性和制度一致性检查。

## 11. 自检

命令：

```text
npm run 自检
```

当前自检目标：

- 同种子复现；不同种子不同。
- 集合竞价拒绝市价单。
- 集合竞价可提交、09:20 后锁定撤单、09:30 形成开盘价。
- 集合竞价订单不丢失，成交或转连续竞价。
- 连续竞价同价订单先到先成交。
- 部分成交后冻结资产正确。
- 撤单和跨日过期后冻结资产释放。
- 交易价与盘口不突破涨跌停边界。
- 现金、份额、订单数量与冻结资产保持一致。

GitHub Actions 工作流已配置，但当前对话尚未取得可靠云端执行结果。不得写成“云端已通过”。
