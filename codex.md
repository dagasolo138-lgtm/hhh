# 中国黄金模拟市场 · Codex

> 项目长期交接中枢。开发时按需要查阅；**每次开发完成后，必须在本文末尾追加版本更新概述。**

## 1. 项目定义

单标的、纯前端、GitHub Pages 中国黄金模拟市场。唯一资产是人民币计价的中国黄金现货ETF模拟份额。

```text
国际金价 × 美元兑人民币 ÷ 31.1034768
→ 人民币换算金价
→ 国内黄金溢价与国内现货
→ 基金理论净值
→ 开盘集合竞价 / 连续竞价中心订单簿
→ 二级市场交易价格
```

项目用于市场机制模拟与教育，不连接真实交易，不构成投资建议。

## 2. 不可破坏的规则

1. 理论净值与二级市场交易价格分离。
2. 技术指标只能影响后续订单，不能直接改价格。
3. 本刻结果只允许在下一刻反馈。
4. 同初始状态、随机种子和调用顺序必须可复现。
5. 自动交易者、做市商与玩家的现金和份额必须守恒。
6. 智能分析只读，不能修改市场、账户、订单或存档。
7. 连续竞价必须价格优先、同价时间优先和自成交保护。
8. 玩家限价单冻结资产必须与待处理剩余数量严格一致。
9. 开盘集合竞价、连续竞价、市价单、限价单、撤单、过期与存档必须可追溯。

## 4. 当前交易架构

### 4.1 开盘集合竞价

```text
09:15–09:20
玩家限价单可提交、可撤销
五类自动资金按目标仓位生成开盘限价单

09:20–09:25
玩家可继续提交，撤单锁定
自动订单按最新目标仓位刷新后锁定

09:30
玩家订单 + 自动订单 + 做市背景深度
→ 单一开盘价
→ 统一清算
→ 玩家剩余订单转入连续竞价中心簿
→ 自动剩余订单登记为连续接续意图
```

开盘价顺序：最大可成交量 → 最小未成交量 → 最接近前收盘价 → 最小报价单位。

- 玩家集合成交会写入玩家可见日志。
- 玩家未成交部分保留订单与冻结资产，转入连续竞价。
- 自动未成交部分在下一刻按连续竞价规则重新报价；当前不保留原开盘限价。

### 4.2 连续竞价中心订单簿

参与者：做市商、价值套利、趋势资金、散户群体、风险控制资金、先行信号资金、玩家市价单和玩家限价单。

```text
买单：价格高者优先；同价按中心委托序号早者优先。
卖单：价格低者优先；同价按中心委托序号早者优先。
```

- 玩家连续限价单在提交瞬间分配中心委托序号。
- 集合竞价转入连续的玩家订单在转入瞬间分配中心委托序号。
- 玩家市价单即时有效；未成交残余当刻失效且从中心盘口快照移除。
- 玩家限价单跨刻保留、当日有效、跨日失效。
- 订单成交直接更新双方现金、持仓、平均成本和已实现盈亏。

## 5. 关键状态容器

```text
结构.交易制度
结构.玩家冻结资产 = { 现金, 份额 }
结构.玩家委托 = { 下次编号, 待处理, 历史, 状态统计 }
结构.集合竞价 = {
  当前交易日, 待处理, 历史,
  自动订单, 自动历史, 自动订单已生成, 自动订单锁定,
  自动转入连续, 最近结果, 最近开盘价, 最近成交量, 最近成交额
}
结构.中心订单簿 = {
  下次委托序号, 本刻订单, 本刻成交, 订单历史, 成交历史,
  盘口快照, 本刻自动意图, 开盘转入订单, 版本, 最近更新
}
```

关键订单字段：

```text
编号、委托序号、中心委托序号、来源、参与者标识、方向、价格、数量、
已成交数量、剩余数量、成交金额、状态、有效期、做市报价、自动生成、玩家订单、市价单
```
